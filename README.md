# FoodNStuff — Meal Planner

Personal meal prep planner. Calories from the German BLS 4.0 food database.

**Live:** https://fhieser.github.io/FoodNStuff/

---

## Quick Start (local dev)

```bash
# One-time setup (first run only)
pip install openpyxl
python scripts/convert_bls.py

# Start the app
python -m http.server 8000
```

Open **http://localhost:8000/web/**

---

## Adding a Meal

1. Create a `.yaml` file in the right folder under `data/meals/`:

```
data/meals/
├── lunch/
├── snacks/
└── prep/
```

2. Push to `main` — GitHub Actions regenerates `web/js/content-index.js` and redeploys automatically.

### Meal YAML format

```yaml
name: My Meal
description: Short description for the card view.
ingredients:
  - name: Hähnchen Brustfilet
    bls_code: V416100
    amount_g: 150
  - name: Reis
    bls_code: C352000
    amount_g: 100
steps:
  - Cook rice
  - Grill chicken
notes: Optional notes shown at the bottom.
```

### Finding BLS Codes

Search [data/ingredients/bls.json](data/ingredients/bls.json) for the ingredient name to get its `bls_code`.
All weights are **raw/uncooked** unless noted otherwise.

## Structure

```
FoodNStuff/
├── .github/
│   ├── scripts/
│   │   └── generate-index.js   ← run by CI to rebuild content-index.js
│   └── workflows/
│       └── deploy.yml          ← GitHub Pages deployment
├── BLS_4_0_2025_DE/            ← source database (do not edit)
├── data/
│   ├── meals/                  ← meal YAML files
│   └── ingredients/
│       └── bls.json            ← generated by convert_bls.py
├── scripts/
│   └── convert_bls.py          ← run once to generate bls.json
└── web/                        ← website
    └── js/
        └── content-index.js    ← auto-generated by CI (do not edit manually)
```
